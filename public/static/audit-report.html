DB Schema Updates (Migration 0008)

-- 0008_phase2_enhancements.sql

-- Async build queue events
CREATE TABLE IF NOT EXISTS build_queue_events (
  id TEXT PRIMARY KEY,
  job_id TEXT NOT NULL,
  event_type TEXT NOT NULL,  -- enqueued, processing, completed, failed
  payload TEXT,              -- JSON
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (job_id) REFERENCES build_jobs(id) ON DELETE CASCADE
);

-- Feature flag per-user overrides
CREATE TABLE IF NOT EXISTS user_feature_flags (
  user_id TEXT NOT NULL,
  flag_key TEXT NOT NULL,
  enabled INTEGER NOT NULL DEFAULT 1,
  PRIMARY KEY(user_id, flag_key),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- Email verification tokens
CREATE TABLE IF NOT EXISTS email_verifications (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  token_hash TEXT UNIQUE NOT NULL,
  expires_at DATETIME NOT NULL,
  used INTEGER NOT NULL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- Monthly grant log (prevent duplicate grants)
CREATE TABLE IF NOT EXISTS coin_grant_log (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  period TEXT NOT NULL,           -- YYYY-MM
  coins_granted INTEGER NOT NULL,
  granted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, period)
);

R-008 Security No email verification required — bot signups possible High Medium P2 Email verification flow (Task 2.6). Restrict build/deploy to verified accounts.
R-009 Infrastructure D1/KV/R2 placeholder IDs in wrangler.jsonc High Blocks production deploy P1 Run: wrangler d1 create deploy-production. Update IDs in wrangler.jsonc. Same for KV + R2.

R-013 2FA two_factor_secret stored as plaintext Medium Relevant at Phase 3 P3 Encrypt TOTP secret with a KV-stored encryption key before writing to D1.

Intent Layer

The Intent Layer is the heart of DEPLOY's AI orchestration. Every AI call passes through AIService.processIntent() — it never executes arbitrary code, only mapped, named intents.

IntentLog (Audit Trail):

Stripe Integration Architecture

Flow: Buy Coins
1. User clicks "Buy Coins"
2. Frontend: GET /api/vault (shows packages)
3. User selects package
4. POST /api/vault/checkout
5. Server: StripeService.createCoinCheckoutSession()
6. Returns: checkout_url
7. Frontend: redirect to Stripe Hosted Page
8. Stripe → webhook POST /api/webhooks/stripe
9. handleEvent('checkout.session.completed')
10. CoinService.credit(userId, coins)
11. billing_events INSERT
12. ResendService sends purchase email
Flow: Plan Upgrade
1. POST /api/vault/checkout-plan
2. ensureStripeCustomer() → create/fetch customer
3. createPlanCheckoutSession(plan.stripe_price_id)
4. Redirect to Stripe subscription checkout
5. Webhook: invoice.payment_succeeded
6. Update memberships.plan_id
7. Grant plan bonus coins
8. Webhook: customer.subscription.deleted
9. Downgrade to free plan
